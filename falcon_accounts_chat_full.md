## المساعد

**فريق الخبراء الافتراضي:** كريم منصور — محاسبة إدارية ودوره ضبط الأرباح ورأس المال، عمر فؤاد — تصميم أنظمة وعمليات ودوره تحويل الرسم لسيستم، سارة نبيل — تحليل بيانات ومبيعات ودورها بناء التقارير والمؤشرات.

## هدف الجلسة

تحويل الرسم المبدئي إلى **هيكل حسابات واضح لفلكون** يجاوب في أي لحظة عن 5 أسئلة:

1. معانا سيولة فعلية كام؟
2. لينا فلوس عند شركة الشحن أو العملاء كام؟
3. علينا فلوس للمطبعة أو الموظفين كام؟
4. كل أوردر كسب أو خسر كام؟
5. الربح القابل للتوزيع على الشركاء كام؟

الرسم عندك فيه أساس ممتاز، خصوصًا:

- فصل رأس مال البضاعة عن الأرباح.
- وجود وارد وصادر.
- تقسيم المصروفات إلى طباعة، شحن، تسويق، مرتبات وغيرها.
- فكرة أن رأس مال التشغيل لا يُمس.

لكن محتاجين نفصل بين **مكان الفلوس** و**سبب حركة الفلوس**؛ لأن دي أكتر نقطة بتبوّظ حسابات المشاريع الصغيرة.

### 1. الهيكل الرئيسي للسيستم

```text
الأوردرات
   ↓
المدفوعات والمحافظ
   ↓
تكلفة الأوردر
   ↓
المخزون والموردون
   ↓
المصروفات
   ↓
الأرباح وحسابات الشركاء
```

> **أي أوردر بدون Order ID يعتبر غير موجود محاسبيًا.**

كل دفعة أو مصروف أو مرتجع لازم يكون مرتبطًا بـOrder ID أو Supplier ID أو Expense ID أو Partner Transaction ID.

### 2. شجرة حسابات فلكون

#### الأصول — Assets

| الحساب | أمثلة |
|---|---|
| خزنة نقدية | كاش موجود فعليًا |
| InstaPay | رصيد التحصيل على إنستاباي |
| Vodafone Cash | رصيد فودافون كاش |
| الحساب البنكي | لو موجود |
| أموال لدى شركة الشحن | أوردرات اتسلّمت ولسه شركة الشحن محوّلتش الفلوس |
| عهدة موظفين | فلوس مع شخص لشراء خامات أو دفع شحن |
| مخزون جرابات | الجرابات الجاهزة أو الخام المملوكة لفلكون |
| مخزون إكسسوارات | PopSocket، Keychain، Screen Protector |
| مخزون تغليف | بوكسات، أكياس قماش، كروت، استيكرات |
| مقدمات للمطبعة | فلوس مدفوعة قبل استلام الطباعة |

#### الالتزامات — Liabilities

| الحساب | مثال |
|---|---|
| مقدمات عملاء | عميل دفع نصف الأوردر ولسه الأوردر لم يُسلّم |
| حساب المطبعة | طباعة اتعملت ولسه لم تُدفع |
| حساب شركة الشحن | رسوم شحن أو مرتجعات لم تُسوَّ بعد |
| مرتبات وعمولات مستحقة | مبلغ لم يُدفع للموظف أو المودريتور |
| قروض الشركاء | شريك أدخل مبلغًا مؤقتًا وليس زيادة رأس مال |

المبلغ الذي يدفعه العميل مقدمًا في أوردر مخصص **ليس ربحًا فورًا**، بل مقدم عميل مقابل أوردر تحت التنفيذ، ويتحول إلى إيراد فعلي عند تسليم الأوردر.

#### رأس المال وحقوق الشركاء

- رأس مال أحمد.
- رأس مال معاذ.
- أرباح محتجزة داخل فلكون.
- مسحوبات أحمد.
- مسحوبات معاذ.
- أرباح موزعة لأحمد.
- أرباح موزعة لمعاذ.
- مبالغ إضافية دخلت كقرض من شريك.

### 3. الإيرادات

| الإيراد | مثال |
|---|---|
| مبيعات جرابات مطبوعة | الجراب بـ160 جنيه |
| مبيعات Custom | التصميمات المخصصة |
| مبيعات الجرابات الجاهزة | Ready Stock |
| مبيعات Screen Protector | مثل Curved Screen |
| مبيعات الإكسسوارات | Keychain، Popsocket |
| رسوم التصميم | لو العميل يدفع مقابل تصميم خاص |
| رسوم الشحن المحصلة | المبلغ الذي دفعه العميل للشحن |
| خصومات المبيعات | تسجل بالسالب |
| مردودات المبيعات | أوردرات تم إلغاؤها أو إرجاعها |

```text
الشحن المحصل من العميل
− تكلفة شركة الشحن
= ربح أو خسارة الشحن
```

### 4. تكلفة الأوردر المباشرة

| البند | القيمة |
|---|---:|
| سعر الجراب الخام | 45 |
| تكلفة الطباعة | 30 |
| البوكس والتغليف | 12 |
| هدية أو Keychain | 8 |
| تكلفة الشحن الفعلية | 65 |
| عمولة دفع أو تحويل | 3 |
| إجمالي تكلفة الأوردر | 163 |

```text
ربح المنتجات = قيمة المنتجات − تكلفة المنتجات والتغليف
نتيجة الشحن = الشحن المحصل − تكلفة الشحن الفعلية
هامش الأوردر = إجمالي الإيراد − إجمالي التكلفة المباشرة
```

### 6. المحافظ المالية المقترحة

| المحفظة | وظيفتها |
|---|---|
| محفظة التحصيل | تدخل فيها كل مدفوعات العملاء |
| محفظة الإنتاج والمخزون | الطباعة وشراء الجرابات والإكسسوارات |
| محفظة التشغيل | مرتبات، إعلانات، برامج، مواصلات |
| احتياطي المرتجعات والطوارئ | إعادة شحن، تلف، أخطاء، تأخير |
| أرباح قابلة للتوزيع | لا تُملأ إلا بعد الإقفال الشهري |

```text
التحصيلات
↓
سداد تكاليف الأوردرات
↓
سداد التزامات المطبعة والشحن
↓
إعادة ملء رأس مال التشغيل
↓
تغطية المصروفات العامة
↓
تكوين احتياطي المرتجعات
↓
المتبقي = صافي الربح القابل للتوزيع
```

بدل حساب «رأس المال يتجدد أسبوعيًا»، نستخدم **الحد الأدنى لرأس مال التشغيل**.

### 7. دورة الأوردر محاسبيًا

| حالة الأوردر | ماذا يحدث ماليًا؟ |
|---|---|
| New Order | لا حركة مالية |
| Waiting Payment | لا حركة حتى وصول دفعة |
| Deposit Paid | تسجل دفعة عميل مقدمة |
| Design Confirmed | تثبيت السعر والتكلفة المتوقعة |
| Sent to Print | تسجيل تكلفة متوقعة على المطبعة |
| Printed | تسجيل التكلفة الفعلية |
| QC Passed | الأوردر جاهز للشحن |
| Shipped | تسجيل الشحنة ورقم التتبع |
| Delivered | إثبات البيع والإيراد الفعلي |
| Courier Settlement | تحويل المستحق من شركة الشحن |
| Returned | عكس البيع وتسجيل تكلفة المرتجع |
| Problem Order | تسجيل سبب المشكلة ومن يتحمل التكلفة |

مثال أوردر Custom:

```text
315 دخلت InstaPay
↓
250 مقدم عميل
65 شحن محصل مقدمًا
↓
بعد التسليم يتحول سعر المنتجات إلى مبيعات
↓
يُخصم المتبقي الذي حصلته شركة الشحن
↓
يظهر هامش الربح النهائي للأوردر
```

## المساعد

**الخبراء المتعاونون:** كريم منصور — محاسبة شركات ودوره تنظيم حقوق الشركاء، عمر فؤاد — تصميم سيستم ودوره تحويل الاتفاق لقواعد تلقائية، سارة نبيل — تحليل مالي ودورها ضبط توزيع الأرباح.

## تم تثبيت النقطة الأولى

- **ملكية أحمد:** 50%
- **ملكية معاذ:** 50%
- **توزيع صافي الأرباح:** 50% لأحمد و50% لمعاذ

الحسابات المستقلة داخل السيستم:

```text
حساب رأس مال أحمد
حساب رأس مال معاذ
حساب مسحوبات أحمد
حساب مسحوبات معاذ
أرباح مستحقة لأحمد
أرباح مستحقة لمعاذ
```

### قاعدة محاسبية مهمة

المساواة في الملكية والأرباح لا تعني بالضرورة تساوي المرتبات.

```text
مرتب أو مقابل عمل لكل شخص
+
توزيع صافي الأرباح 50/50
```

المرتب مقابل العمل اليومي، أما الأرباح فمقابل الملكية والمخاطرة.

مثال عند وجود صافي ربح قابل للتوزيع قدره 20,000 جنيه:

| الشريك | النسبة | الأرباح المستحقة |
|---|---:|---:|
| أحمد | 50% | 10,000 |
| معاذ | 50% | 10,000 |

لو أحمد سحب 6,000 فقط، يظل له 4,000 أرباح مستحقة، ولا تسجل الـ6,000 كمصروف على فلكون.

### السؤال الثاني

فلوس فلكون حاليًا تدخل فين؟

1. حسابات ومحافظ شخصية باسم أحمد أو معاذ.
2. بعضها شخصي وبعضها مخصص لفلكون.
3. كلها في حساب أو محافظ منفصلة لفلكون.

### أهم 3 أخطاء أو فخاخ

1. تسجيل مسحوبات الشريك كمصروف للشركة.
2. خلط المرتبات بتوزيع الأرباح.
3. تقسيم رصيد الخزنة قبل خصم الالتزامات.

### تطوير على 3 مستويات

- **Quick Win:** حساب مستقل لكل شريك ومسحوباته وأرباحه.
- **Medium:** إقفال شهري وتقسيم تلقائي 50/50.
- **Advanced:** حساب جاري لكل شريك يجمع رأس المال والقروض والأرباح والمسحوبات.

### Glossary

- **Partner Capital:** رأس مال الشريك.
- **Drawings:** مسحوبات شخصية.
- **Partner Current Account:** الحساب الجاري للشريك.
- **Retained Earnings:** أرباح محتجزة.

```text
الملكية: 50/50
↓
المرتبات منفصلة عن الملكية
↓
إقفال الحسابات
↓
تحديد صافي الربح الحقيقي
↓
توزيع الأرباح 50% أحمد و50% معاذ
```

### هيكل المحافظ

حتى لو توجد محفظة فعلية واحدة، يتم تقسيم رصيدها داخليًا:

| الرصيد الداخلي | مثال |
|---|---:|
| أموال مخصصة للطباعة | 7,000 |
| مقدمات عملاء | 3,000 |
| تشغيل وشحن | 4,000 |
| احتياطي مرتجعات | 2,000 |
| أرباح غير موزعة | 4,000 |

3. جعل البونص رأيًا شخصيًا بدل الاعتماد على بيانات فعلية.

### تطوير على 3 مستويات

- **Quick Win:** تسجيل الرواتب والبونص والموافقات يدويًا.
- **Medium:** Score تلقائي، كشف راتب، منع الخصم الخاسر، وتجميع السحوبات.
- **Advanced:** توقع البونص والسيولة، وتحليل الأخطاء والأداء.

### Glossary

- **Salary Window:** فترة دفع الراتب.
- **Due Date:** تاريخ الاستحقاق.
- **Approval Threshold:** حد الموافقة.
- **Negative Margin:** هامش سالب.
- **Bonus Range:** نطاق البونص.
- **Scorecard:** بطاقة الأداء.
- **KPI:** مؤشر أداء رئيسي.
- **ERD:** مخطط علاقات قاعدة البيانات.

### تطوير على 3 مستويات

- **Quick Win:** العملاء، الأوردرات، العناصر، المدفوعات، الطباعة، الشحن، المصروفات، والمحفظة.
