# Arabic UX Content Guide

- Product copy is Arabic, concise, and task-oriented; API identifiers remain English.
- The document uses `lang="ar"` and `dir="rtl"`.
- Use `أوردر` only where current staff terminology benefits recognition; accounting terms use formal Arabic.
- Display statuses as source values until an approved status dictionary supplies an unambiguous translation.
- Errors say what failed and confirm that no change was made; never display a success message before the RPC succeeds.
- Dates use Cairo time. Money displays `ج.م` with Arabic digits and two exact minor-unit digits.
- IDs, references, emails, and correlation identifiers use isolated LTR direction.
