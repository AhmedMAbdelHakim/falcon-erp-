# دليل مسؤول النظام

تعيينات المستخدمين والأدوار تكون في قاعدة البيانات وبنطاق مؤسسة وفترة فعالية. لا تستخدم `raw_user_meta_data` لمنح الصلاحيات. راجع فصل الواجبات قبل منح أدوار المالية أو الشركاء أو الإقفال.

تحقق دوريًا من المستخدمين غير النشطين، التعيينات المنتهية، الصلاحيات الحساسة، سجلات الرفض، مفاتيح البيئة، وسياسات Storage. لا يمنح دور مسؤول النظام موافقة مالية بشرية بدلًا من صاحب المسؤولية.

أي تغيير إعداد مالي يحتاج سببًا، نسخة فعالة، مراجعة، وأثر تدقيق. استخدم إجراء الطوارئ فقط عند حادث موثق.
